Как работает пломба — защита от подделки
4 шага от создания до проверки на сдаче
1
Волонтёр сканирует ID студента на пункте выдачи
Система генерирует уникальные ID для каждой коробки
2
Принтер печатает QR: номер + ФИО + дата + хэш
Хэш = SHA256(ID студента + дата + секретная соль сервера)
3
Наклейка клеится на коробку, сверху пломба-стикер
Пломба одноразовая — при вскрытии деформируется
4
При сдаче волонтёр сканирует — система проверяет хэш
Несовпадение = красный флаг в журнале нарушений

Критические риски и протоколы решения
Все сценарии сбоя — с готовыми ответами
✕
Нет интернета на пункте выдачи
→ Offline-режим с локальной синхронизацией; резервный бумажный журнал с теми же полями
⊘
Студент скопировал чужой QR-код
→ Хэш включает ID + дату + соль — копия не пройдёт проверку, система покажет ошибку
◌
QR-наклейка отклеилась или повреждена
→ Дублирующий бумажный ярлык внутри каждой коробки с тем же номером
◷
Студент не пришёл в забронированный слот
→ Авто-напоминание за 2 ч и 30 мин; слот автоматически уходит в очередь ожидания
◫
Коробка потерялась при транспортировке
→ Фото при погрузке + при разгрузке; страховое фото на накопительном пункте при сдаче
